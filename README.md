# Azure LLM Council (AI 위원회)

이 프로젝트는 단일 AI 모델의 한계를 극복하기 위해, 여러 개의 최첨단 AI 모델들(GPT-5, GPT-4, Claude Opus 등)이 모여 **"위원회(Council)"**를 구성하고, 서로 토론하고 검증하여 최적의 답변을 도출하는 시스템입니다.

## 💡 이 소스가 왜 필요한가요? (Why This Matters)

AI를 사용하다 보면 가끔 엉뚱한 대답을 하거나(Hallucination), 한쪽으로 치우친 정보를 줄 때가 있습니다. 이 프로젝트는 다음과 같은 이유로 더 신뢰할 수 있는 답변을 제공합니다:

1.  **집단지성 (Collective Intelligence)**
    *   한 명의 천재보다 여러 명의 전문가가 낫습니다. OpenAI의 GPT 계열과 Anthropic의 Claude 계열 등 서로 다른 특성을 가진 모델들이 각자의 관점에서 문제를 바라봅니다.

2.  **상호 검증 (Cross-Validation)**
    *   모델들이 서로의 답변을 "심사위원" 자격으로 평가합니다. 서로의 답변에 대해 순위를 매기고, 장점과 단점을 지적하는 과정을 통해 잘못된 정보가 자연스럽게 걸러집니다.

3.  **최고의 답변 종합 (Synthesis)**
    *   단순히 여러 답변을 나열하는 것이 아닙니다. "의장(Chairman)" 역할을 맡은 모델이 모든 답변과 평가(비평)를 종합하여, 각 답변의 장점만을 모은 **최종 결론**을 내립니다.

---

## 🚀 작동 원리 (Workflow)

이 시스템은 3단계의 회의 과정을 거칩니다:

### 1단계: 의견 제시 (Stage 1)
사용자가 질문을 던지면, 5명의 위원(모델)들이 각자 독립적으로 답변을 작성합니다.
*   참여 모델: `gpt-5`, `gpt-5.1`, `gpt-4.1`, `claude-opus-4-5`, `claude-sonnet-4-5`

### 2단계: 상호 평가 (Stage 2)
모든 모델이 다른 모델들의 답변을 익명으로 검토합니다.
*   **랭킹 선정:** 가장 정확하고 통찰력 있는 답변 순서대로 순위를 매깁니다.
*   **코멘트 작성:** 각 답변의 잘된 점과 부족한 점을 한국어로 비평합니다.

### 3단계: 최종 판결 (Stage 3)
**의장(Chairman, GPT-5.1)**이 1단계의 모든 답변과 2단계의 모든 평가 내용을 검토합니다. 이를 바탕으로 가장 완벽하고 균형 잡힌 **최종 답변**을 작성하여 사용자에게 제공합니다.

## 🛠️ 설치 및 실행 방법

### 1. 환경 변수 설정 (.env)
프로젝트 루트에 `.env` 파일을 생성하고 다음 정보를 입력해야 합니다. (Azure OpenAI 및 Azure Anthropic 설정)

```ini
# 공통 API 키 및 엔드포인트
OPEN_AI_KEY_5=your_api_key_here
OPEN_AI_ENDPOINT_5=https://your-resource-name.openai.azure.com

# (선택) 모델 배포 이름 설정 (기본값이 코드에 지정되어 있음)
# AZURE_DEPLOY_GPT5=gpt-5
# AZURE_DEPLOY_GPT5_1=gpt-5.1
# ...
```

### 2. 실행
파이썬 스크립트를 실행하면 질문을 입력받는 프롬프트가 뜹니다.

```bash
python Azure_LLM_council.py
```

### 3. 결과 확인
콘솔 창에서 각 단계별 진행 상황(각 모델의 답변 -> 상호 평가 결과 -> 최종 답변)을 실시간으로 확인할 수 있습니다.
